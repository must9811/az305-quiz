# 基本設計書

## 1. 文書概要

### 1.1 目的
- `udemy/問題集1.html` に保存された Udemy 問題集の結果画面から、問題文・選択肢・正答・解説・画像参照を抽出し、HTML/CSS/JavaScript で動作する単一ページのクイズアプリを構築するための基本設計を定義する。

### 1.2 対象範囲
- 実装対象データは `udemy/問題集1.html` のみとする。
- `問題集1.html` に含まれる 60 問を、抽出時点の固定順で 10 問ずつ 6 セットに分割する。
- 将来、`udemy` 配下の他の問題集 HTML を同じ方式で追加できる構成を前提にする。

### 1.3 前提
- アプリは静的ファイルのみで構成し、バックエンドは使用しない。
- 進捗、マーク、直近結果はブラウザの `localStorage` に保存する。
- セット内容とセット内の出題順は固定であり、v1 ではランダム出題を行わない。

## 2. 入力データの前提

### 2.1 入力元
- HTML: `udemy/問題集1.html`
- 画像: `udemy/問題集1_files/*`

### 2.2 入力データの確認結果
- 総問題数: 60 問
- セット分割: 10 問 x 6 セット
- 画像付き問題: 43 問
- 複数選択問題: 4 問
- ケーススタディを含む問題: 6 問
- 選択肢数は固定ではなく、2 個から 6 個まで混在する

### 2.3 抽出ルール
- 問題文: `id="question-prompt"` を持つブロックから取得する
- 選択肢: `data-purpose="answer"` を持つブロックから取得する
- 正答判定: `正解` または `正しい選択` ラベルを持つ選択肢を正答とする
- 解説: `id="overall-explanation"` を持つブロックから取得する
- 画像: 問題文・解説に含まれる相対 `img src` を取得する

### 2.4 抽出時の補足ルール
- 「前の問題と2問1セット」「次の問題と2問1セット」「ケーススタディ」などの関連性は `contextGroupId` で保持する
- v1 では設問本文の重複除去は行わない
- 分野ラベルは抽出できる場合のみ保持し、取得できない場合は `null` とする

## 3. システム構成

### 3.1 構成方針
- 単一ページアプリケーション (SPA) とする
- 問題データは事前抽出済み JSON を読み込む
- UI 状態遷移は JavaScript で管理する

### 3.2 想定ファイル構成
```text
/
  requirements.md
  基本設計書.md
  実装計画書.md
  index.html
  css/
    app.css
  js/
    app.js
    storage.js
    quiz.js
    report.js
    copy.js
  data/
    quiz-manifest.json
    sets/
      set-001.json
      set-002.json
      set-003.json
      set-004.json
      set-005.json
      set-006.json
  scripts/
    extract-quiz-data.js
  udemy/
    問題集1.html
    問題集1_files/
```

### 3.3 データフロー
1. `scripts/extract-quiz-data.js` が `udemy/問題集1.html` を読み込む
2. 問題データを抽出して 60 問の配列を生成する
3. 抽出結果を 10 問単位で 6 セットに分割する
4. `data/quiz-manifest.json` と `data/sets/<setId>.json` を出力する
5. `index.html` 起動時に manifest を読み込み、セット選択画面を表示する
6. セット開始時に該当セット JSON を読み込み、セッション状態を生成する
7. 回答・マーク・レポートを `localStorage` に保存する

## 4. 画面設計

### 4.2 クイズ画面

  - 現在のセット名

  - 問題文
  - 問題内画像

  - 正誤表示

### 4.3 結果レポート画面

## 5. 機能設計

### 5.1 セット選択
- 10 問固定のセットを一覧表示する
- セットごとに開始モードを指定して開始する
- 未完了セッションがある場合は再開導線を先頭に表示する

### 5.2 通常モード
- セット内 10 問を固定順で出題する
- 1 問回答するごとに即時採点する
- 採点直後に解説を表示する

### 5.3 マークのみモード
- 選択セット内で `marked=true` の問題のみを対象にセッションを生成する
- 該当問題が 0 件の場合は開始せず、メッセージを表示する

### 5.4 直近誤答のみモード
- 選択セット内で、直近完了セッション時に誤答だった問題のみを対象にセッションを生成する
- 判定対象は選択した 10 問セット単位とする
- 該当問題が 0 件の場合は開始せず、メッセージを表示する

### 5.5 セッション再開
- `az305.quiz.session.v1` に未完了セッションを 1 件だけ保持する
- 起動時に未完了セッションが存在する場合、セット選択画面に再開カードを表示する
- 再開時は保存された `setId`、`mode`、`questionIds`、`currentIndex` を復元する

### 5.6 マーク機能
- 各問題でマークの ON/OFF を切り替えられる
- マーク状態は問題単位で永続化する

### 5.7 Markdown コピー
- 問題単位で Markdown をクリップボードにコピーする
- 出力形式は以下で固定する

```md
## 問題{sourceQuestionNumber}

{問題文プレーンテキスト}

### 選択肢
- {選択肢1}
- {選択肢2}

### 正答
- {正答1}

### 解説
{解説プレーンテキスト}
```

## 6. データ設計

### 6.1 Manifest
```json
{
  "sourceHtml": "udemy/問題集1.html",
  "generatedAt": "2026-03-31T00:00:00+09:00",
  "setCount": 6,
  "sets": [
    {
      "setId": "set-001",
      "title": "問題集1 1-10",
      "sourceHtml": "udemy/問題集1.html",
      "questionCount": 10,
      "questionNumberRange": [1, 10],
      "hasImages": true,
      "hasMultiSelect": false
    }
  ]
}
```

### 6.2 Set JSON
```json
{
  "setId": "set-001",
  "title": "問題集1 1-10",
  "sourceHtml": "udemy/問題集1.html",
  "questions": []
}
```

### 6.3 Question Schema

| 項目 | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `questionId` | string | 必須 | 一意な問題 ID。例: `q-001` |
| `sourceQuestionNumber` | number | 必須 | 元 HTML 上の問題番号 |
| `promptHtml` | string | 必須 | 問題文の HTML |
| `promptText` | string | 必須 | 問題文のプレーンテキスト |
| `choices` | `QuizChoice[]` | 必須 | 選択肢配列 |
| `correctChoiceIds` | string[] | 必須 | 正答選択肢 ID 配列 |
| `explanationHtml` | string | 必須 | 解説 HTML |
| `explanationText` | string | 必須 | 解説プレーンテキスト |
| `images` | string[] | 必須 | 問題文・解説内の画像パス |
| `domain` | string \| null | 必須 | 分野名。未取得時は `null` |
| `questionType` | string | 必須 | `single` / `multi` / `boolean` |
| `contextGroupId` | string \| null | 必須 | 2問1セットやケーススタディの関連 ID |
| `tags` | string[] | 必須 | 補助タグ。例: `["case-study"]` |

### 6.4 Choice Schema

| 項目 | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `choiceId` | string | 必須 | 例: `q-001-c1` |
| `label` | string | 任意 | 選択肢記号。元データにない場合は `null` 可 |
| `text` | string | 必須 | プレーンテキスト |
| `html` | string | 必須 | 選択肢 HTML |

### 6.5 questionType 判定
- `correctChoiceIds.length > 1` の場合は `multi`
- 選択肢が 2 件かつ「目標を満たしていますか」系の正誤問題は `boolean`
- それ以外は `single`

### 6.6 contextGroupId 付与ルール
- 問題文に「前の問題と2問1セット」「次の問題と2問1セット」がある場合は連番で同じ `contextGroupId` を付与する
- 問題文に「ケーススタディ」がある場合は同一ケースに属する問題へ同じ `contextGroupId` を付与する
- 判定不能な場合は `null`

## 8. フロントエンド API 設計

### 8.1 抽出入力
- `sourceHtmlPath: string`

### 8.2 抽出出力型
- `QuizManifest`
- `QuizSet`
- `QuizQuestion`
- `QuizChoice`

### 8.3 フロント状態型
- `QuestionState`
- `ActiveSession`
- `SessionReport`

### 8.4 主要関数
- `loadManifest()`
- `loadSet(setId)`
- `startSession(setId, mode)`
- `resumeSession()`
- `submitAnswer(questionId, choiceIds)`
- `toggleMark(questionId)`
- `finishSession()`
- `copyQuestionAsMarkdown(questionId)`

## 9. エラー処理方針
- Manifest または Set JSON の読み込み失敗時は、アプリ内エラーメッセージを表示する
- `localStorage` の JSON が破損している場合は該当キーを破棄して初期状態に戻す
- `マークのみ` または `直近誤答のみ` の対象が 0 件の場合は、開始せず空状態メッセージを表示する
- 画像パスが見つからない場合は画像を表示せず、テキストのみ表示する

## 10. 非機能要件
- 静的配信で動作すること
- デスクトップとモバイルの両方で閲覧可能であること
- 問題画面の主要操作は 1 画面内で完結すること
- 保存済み HTML に依存するため、抽出ロジックは `問題集1.html` の DOM 構造変化に弱いことを前提とし、失敗時の検出ログを持てる設計にする

## 11. 受け入れ条件
- `問題集1.html` から 60 問を抽出できる
- 10 問 x 6 セットに分割できる
- 複数選択問題が `correctChoiceIds` の複数要素として保持される
- 画像付き問題で画像参照切れが発生しない
- 1 問ごとに採点と解説表示ができる
- マーク状態が保存され、`マークのみ` に反映される
- `直近誤答のみ` が選択セット単位で機能する
- 途中終了後に同じ位置から再開できる
- レポート画面で結果概要と問題別結果を確認できる
- 問題単位 Markdown コピーが利用できる
