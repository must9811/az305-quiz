# 実装計画書

## 1. 目的
- `udemy/問題集1.html` を入力として、10 問単位の固定セットで学習できるクイズアプリを構築する。
- 本書では、実装順序、成果物、確認内容、リスクを定義する。

## 2. 実装方針
- 先に抽出処理と JSON 仕様を確定し、その後 UI を構築する。
- データ抽出と UI 実装を分離し、将来の問題集追加時は抽出対象 HTML の追加だけで済む構成にする。
- v1 は `問題集1.html` のみを受け入れ対象とし、抽出結果と UI の整合性確認を優先する。

## 4. フェーズ計画

### フェーズ1: 抽出 PoC
目的:
- `問題集1.html` から必要データが静的に取得できることを確認する

実装内容:
- HTML を読み込む抽出スクリプトを作成する
- 問題文、選択肢、正答、解説、画像参照を抽出する
- 60 問抽出できることを確認する

完了条件:
- 問題数 60 件
- 画像付き問題の画像パス取得
- 複数選択問題の正答複数取得

確認項目:
- 問題番号が 1 から 60 まで連続していること
- 4 問の複数選択問題が複数正答として抽出されること

### フェーズ2: JSON 仕様固定
目的:
- 再利用可能なデータ形式へ整理する

実装内容:
- `QuizManifest` と `QuizSet` の JSON 形式を確定する
- 60 問を 10 問単位で 6 セットへ分割する
- `questionType`、`contextGroupId`、`tags` を付与する

完了条件:
- `data/quiz-manifest.json` を生成できる
- `data/sets/set-001.json` から `set-006.json` を生成できる

確認項目:
- 各セットが 10 問固定であること
- セット内の順序が元 HTML の番号順と一致すること
- `domain` が未取得時に `null` になること

### フェーズ3: SPA 骨格とセット選択画面
目的:
- アプリの起点画面を構築する

実装内容:
- `index.html` と基本レイアウトを作成する
- Manifest 読み込み処理を実装する
- セット一覧と開始モード選択 UI を実装する
- 再開セッションカード表示を実装する

完了条件:
- アプリ起動時に 6 セットが選択画面へ表示される
- `通常`、`マークのみ`、`直近誤答のみ` の導線がある

確認項目:
- 未完了セッションがない場合は通常一覧のみ表示されること
- 未完了セッションがある場合は再開カードが最上部に表示されること

### フェーズ4: 通常モード出題

- 回答確定

- 解説表示
- 固定順遷移

完了条件:
- 通常モードで 10 問を最後まで実施できる

確認項目:
- 1 問ごとに正誤判定が表示されること
- 解説が採点後に表示されること

## 7. リスクと対策

### 7.1 DOM 構造依存
- リスク: Udemy 保存 HTML の DOM 構造が問題集ごとに微妙に異なる可能性がある
- 対策: `data-purpose` と結果ラベルを優先し、クラス名依存を最小化する

### 7.2 分野ラベル不足
- リスク: 問題単位の分野ラベルが HTML から取得できない場合がある
- 対策: `domain` は `null` を許容し、v1 の UI では必須表示にしない

### 7.3 画像パス切れ
- リスク: 抽出後の表示パスと実ファイル位置がずれる可能性がある
- 対策: 元 HTML に対する相対パスを基準に保持し、表示時にそのまま利用する

### 7.4 保存データ破損
- リスク: `localStorage` の JSON 破損で起動不能になる可能性がある
- 対策: 読み込み時に例外処理を行い、破損キーのみ初期化する
